MPS Career (仮)

MPS Career (仮) は、MPS のプロジェクトや勉強会で得た知識や技術を、 さらなるキャリア形成のために 「実際に報酬の発生する仕事で実践する場」として MPS の運用に必要なシステム構築の一部または全部をお仕事として依頼する場です。

Light weight projects

MPS Career ではシステム構築を忙しい生活の中でも頑張ればこなせる程度の粒度のコンポーネントに分解し、 お仕事としてご依頼します。お仕事を請け負う方のレベルによりかかる日数は異なりますが、多くの場合、 プロが行った場合には、1、2日で終わる分量の仕事になっています。

Remote works

お仕事は基本リモートで結構です。また、MPS Atelier がオープンしており、イベントが開催されない時間には、 仕事場として MPS Atelier お使いいただけます。

Open to public

ほとんどのお仕事は、オープンソースで Github 上に公開されます。 GitHub での push & pull 履歴はご自身のチーム開発の実績として存分にお使いください。

How it works

  1. GitHub にアカウントを作成 (すでにお持ちの方は不要です)
  2. MPS Career に GitHub アカウントでログイン
  3. お仕事を検索
  4. 気に入ったお仕事にエントリー
  5. 必要に応じて MPS Atelier にて面談
  6. お互いに条件が合えばお仕事決定!
  7. 開発 & 定期ミーティング
  8. 納品
  9. 品質チェック
  10. 品質チェック通過
  11. 報酬受け取り

Policies of MPS Career

MPS Career は MPS や自主学習で得た知識や技術を「報酬の発生する仕事で実践する」ことによって、さらなるキャリアアップを目指すための場です。

下記ポリシーはその精神に則って設定されるものです。

Recruitment Policies

MPS Career は MPS や自主学習で得た知識や技術を「報酬の発生する仕事で実践する」ことによって、さらなるキャリアアップを目指すための場です。 そのため、エントリー時の技術的能力よりも、新しい知識や技術を習得しようとする気持ちと、それを完遂できる能力を重視します。

どのような知識や技術 (たとえばフレームワークやライブラリ) を用いて仕事をしたいか、ということについては柔軟に対応します。 「すでに仕事で使っている慣れた技術」も結構ですが「これから仕事などで使ってみたい技術」を積極的に申し出て練習していただけると嬉しいです。

Code Acceptance Policies

プログラム開発の仕事をする上で金子が重要視するのは、自分の作ったコードの構造をきちんと論理的に説明できる、ということです。 この点をクリアできないものについてはまず報酬は支払われません (「え、でも動くじゃん」は認められません)。 この能力は技術者として生きていくのには必須だと考えていますので、クリアできるまで何度でも指摘はします。 いい練習の場だと思っていただけると幸いですし、その程度のガッツのある人を募集します。

MPS について

MPSは、「誰でもリーダーになることができる、社会に役立つIoTやソフトウェアの開発」を中心としたプロジェクトの運営を通して、 実践的な知識と技術の獲得だけでなく、高いリーダーシップ、メンバーシップを有した世の中を牽引する人材の育成を行います。 また、多くのプロジェクトをみんなで楽しみながら遂行し、見える実績と強固な人脈を作ることにより、メンバーのキャリアチェンジやキャリアアップの礎を築きます。
